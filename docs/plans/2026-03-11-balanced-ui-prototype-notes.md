# Balanced UI Prototype Notes

日期：2026-03-11
状态：Applied

## 这版原型的目标

把产品收敛到一个更稳的平衡点：

- 不是纯协作可视化工具
- 不是纯公司模拟 / 游戏界面
- 而是一个 `能真实交付成果，同时让用户感到自己在经营一家公司` 的老板工作台

---

## 这版刻意保留的三层结构

### 第一层：老板主视角

首页先回答：

- 今天只看什么
- 现在该拍板什么
- 哪些事不用现在看

这是为了压低心智负担。

### 第二层：秘书中间层

会议、分歧、风险不会直接扑到首页，  
而是先经过秘书长整理成：

- 老板简报
- 决策包
- 会议纪要
- 风险升级单

这是为了避免产品滑回“群聊管理器”。

### 第三层：组织活性背景层

公司仍然要表现出“在活着”，但不通过沙盘或实时拓扑去表现。

所以这版只保留轻量信号：

- 组织协作正在改善
- 市场反馈正在形成
- 制度经验正在沉淀

这部分存在，但默认退到背景层。

---

## 相比上一版的核心变化

### 变化 1：首页主叙事更单一

从“多个模块并排出现”，收敛为：

- `今天只看这三件事`
- `待你拍板`
- `公司正在活着，但不需要你现在介入`

### 变化 2：交付与资产更明确

不仅展示交付物，还开始展示：

- 资产库新增
- 可复用的研究 / 监控沉淀

### 变化 3：经营感与世界感不再抢中心

世界感仍然存在，  
但不再通过：

- Agent 拓扑
- 实时日志
- 戏剧化冲突

去争夺首页主舞台。

---

## 我对这版的判断

如果把产品方向理解成一个比例：

- `70% 真实交付系统`
- `30% 公司经营模拟感`

那么这版已经明显更接近这个比例。

后面继续演进时，优先原则应保持不变：

## `先老板，后组织；先资产，后过程；先决策，后涌现。`
